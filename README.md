# Guide
## การกำหนดปัญหา
พี่ไม่แน่ใจว่าน้องทำแบบไหน แต่เดาว่าน่าจะทำเป็น **Classification** ไม่ **Binary Class** ก็ **Multiple Class** อีกอันหนึ่งเผื่อน้องสนใจคือ **Multi Label**\
**Binary Class** จะเป็นการตัดสินใจโดยมีแค่ 2 class ให้เลือก เช่น เลือกระหว่างหมากับแมว\
**Multiple Class** จะเป็นการตัดสินใจโดยมีมากกว่า 2 class\
**Multilabel** จะเป็นการตัดสินใจมี Targets มากกว่า 1 แต่มี 2 class\

## การจัดการข้อมูล
อันนี้ไม่ค่อยแน่ใจเพราะพี่ไม่รู้ว่าน้องใช้ข้อมูลเป็นเสียงอะไร แต่ได้ยินมาว่าน้องจะไม่จัดการกับ noise เลยเพื่อให้สามารถใช้ในชีวิตจริงได้ แต่อาจจะมีกรณีที่ห้องอัดเสียงมีความเงียบจน noise มีน้อยมาก ในกรณีแบบนี้โมเดลเราอาจเกิด error ได้ ถ้าน้องอยากลองก็ลองทำ model หลายรูปแบบดู แบบหนึ่งมี noise อีกแบบไม่มี noise หรือถ้าอีกรูปแบบหนึ่งคือใช้โมเดลเดียวเลย แต่ว่าจะเป็นการเพิ่มชุดข้อมูลมาอีกชุดหนึ่ง คือเราจะเอาข้อมูลชุดเดิม คัดลอกมันมา และชุดที่คัดลอกมาก็ทำให้ไม่มี noise แล้วค่อยเอาไปเทรนก็ได้ อันนี้แล้วแต่น้องออกแบบเลย

## การสร้างโมเดล
พี่ไม่แน่ใจว่าเราตัดสินใจจะสร้างกี่โมเดล แต่ลองดูอันนี้เป็นไกด์ไลน์ก็ได้ เผื่อช่วยในการตัดสินใจ
